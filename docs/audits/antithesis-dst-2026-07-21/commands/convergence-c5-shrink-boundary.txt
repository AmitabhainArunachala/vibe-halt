## Shape

- workloads::by_name_capturing + PlanCell (Rc<RefCell<Option<FaultPlan>>>):
  KvDemo clones its single retrieved plan into the cell — purely
  observational, no draw/order/trace change. Capture support v1: demo,
  demo-buggy. Runtime-path workloads (disk/net/corpus) retrieve their
  plan INSIDE UniverseCtx::runtime and never hold it — capture there
  needs a kernel API; recorded as a discovered coupling in the ledger,
  surfaced to the user as a typed unsupported diagnostic (exit 2).
- vh_cli::shrink_cli::shrink_universe (pure module; main.rs prints):
  baseline run-twice self-agreement, then try_shrink with the
  exact-fingerprint oracle vh-shrink demands — a candidate passes only
  if two override replays (run_universe_with_fault_plan) agree AND
  reproduce the baseline's exact always-failure (name+detail) set with
  a valid completion. Never any-failure (cause-switch hazard).
- vh run --shrink: additive; minimizes the FIRST failing universe;
  verdict and exit code unchanged; anchored lines `shrink: MINIMIZED`,
  `shrink: UNAVAILABLE`, `shrink-binding:`.
- vh shrink --workload W --seed N --universe K: exit 0 MINIMIZED /
  1 nothing-to-shrink / 2 usage-or-unsupported.
- Provenance binding printed at the boundary (PR #2's open contract —
  ShrinkReport deliberately carries no source/seed binding): workload,
  seed, universe, palette, baseline trace hash, plan digest,
  fingerprint names. Moves into C4 bundles once #19 lands (§5 coupling).
